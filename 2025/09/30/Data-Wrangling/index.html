Data Wrangling

Posted on 2025-09-30 Edited on 2025-10-08

 Data Wrangling is the end-to-end process of inspecting, cleaning, transforming, joining and validating raw or messy data to produce datasets ready for analysis or modeling.

Acquiring Data

 Data acquisition means bringing raw data from various sources into your environment for cleaning and analysis. Although it looks simple, it is often the messiest step because data may come in different formats, systems, or inconsistent states.

 Common sources include files (CSV, Excel, JSON, Parquet), databases (MySQL, PostgreSQL, SQLite), APIs, web scraping, big data platforms (HDFS, Spark, cloud storage), and streaming systems (Kafka, MQTT).

Different File Type Reading Example

Database Reading Example(SQL + Python)
 You can query relational databases using libraries like sqlite3 or SQLAlchemy and then load results into pandas.

API call/Scraping
 APIs are used to fetch data over HTTP (often in JSON format). Python’s requests library can be used to call them. When APIs are not available, scraping can be used to extract data directly from HTML.

Big Data/Data Streaming
 When datasets reach millions/billions of rows or TB/PB size, pandas (which loads everything into memory) becomes insufficient. Distributed computing frameworks (Spark, Dask, Hadoop) or streaming frameworks (Kafka, Flink) are required.

Structuring Data

 Structuring data means converting raw, semi-structured, or unstructured data (JSON, logs, scraped data, text) into clean, well-defined tabular formats suitable for analysis or modeling
